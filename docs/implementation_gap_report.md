# BCNOFNe v3.3.1 実装ギャップ監査報告書 (Implementation Gap Report)

本ドキュメントは、現行の BCNOFNe システムと、マスターが提示した「安定性・保守性・自律性・安全性」強化後の理想状態との差分を洗い出したものです。

## 1. LLM 基盤の統一状況 (LLM Foundation)

### [!] 課題: ハードコードプロンプトの残存
- `core/main.py` の 565行目付近に proactive 思考用のプロンプトが埋め込まれたままになっています。
- `prompts/manifest.yaml` にタスク定義はあるものの、一部のサービスで詳細なパースロジックやリトライポリシーが最適化されていません。

### [!] 課題: プロバイダ設定の分散
- `OpenAIProvider` と `OllamaProvider` のタイムアウトやリトライ設定がコード内に散在しており、一元管理できていません。
- 7B モデル特有の「不完全な JSON」に対する修復ログが不足しており、デバッグが困難です。

---

## 2. 7層メモリシステムの具体化状況 (Memory Service)

### [!] 課題: メモリレイヤーの形骸化
- 仕様上は7層（WORKING, EPISODIC, SEMANTIC, PROCEDURAL, REFLECTIVE, RELATIONAL, MISSION）を謳っていますが、実装上は主に `EPISODIC` に偏っています。
- **昇格条件**: 現在は `importance >= 4` で HDD に送るだけの単純なロジックです。
- **想起ロジック**: 単純なキーワード検索または全件取得に近く、関連性 (Relevance) や感情重み (Emotional Weight) による優先順位付けが未実装です。
- **移送・圧縮**: SSD から HDD への自動移送や、古い記憶の要約・圧縮ロジックが具体化されていません。

---

## 3. 航海日誌の高度化 (Diary Service)

### [!] 課題: 単純な要約に留まっている
- 現状は「直近 100 件のログを投げて要約する」だけであり、以下の要素が欠落しています。
    - 重要イベントの抽出
    - 未完了タスクの特定
    - 次の航海（アクション）の提案
- 出力が LLM の生成結果に依存しすぎており、失敗時のフォールバックが「エラーメッセージ表示」のみで不親切です。

---

## 4. 自律開発 (dev-agent) の安全境界 (Dev Agent)

### [!] 課題: 段階分離と安全性の不足
- 現在も `Observe -> Plan -> Build -> Test -> Apply` の流れはありますが、各段階の構造化ログが不十分です。
- **安全ブレーキ**: 危険操作（大量削除など）の判定ロジックが未実装です。
- **検証の弱さ**: 単純な `py_compile` と `import` テストのみであり、システム全体のスモークテストやユニットテストとの連携が弱いです。
- **ドライラン**: 実際にファイルを書き換える前の「シミュレーション結果」をマスターに提示する仕組みが弱いです。

---

## 5. 監視と観測性 (Watchdog & Health)

### [!] 課題: 6時間間隔による異常検知の遅れ
- フリーズ誤検知防止のために間隔を 6時間 に広げた結果、本当の沈没（サービス停止）に気づくのが遅れるリスクがあります。
- `heartbeat`（生存信号）の概念がなく、パッシブな監視に留まっています。
- `/health` エンドポイントの内容が「ok」のみのサービスが多く、DB 接続や LLM 到達性などの詳細情報が不足しています。
- **監視対象の不足**: `ollama`, `discord-gateway`, `gui`, `oled-controller` などが含まれていません。

## 6. データベースとメモリモデル (Database & Models)

### [!] 課題: テーブルはあるがロジックが未実装
- `InternalStateHistory` や `GoalHistory` テーブルは存在しますが、`core` や `dev-agent` からの活用が十分ではありません。
- `Memory` テーブルに `importance` や `layer` はありますが、前述の通りそれらを判定・活用する AI ロジックが不足しています。

## 7. 結論と優先順位
1.  **最優先**: `watchdog` の heartbeat 併用と、`LLMExecutor` のログ・設定強化。
2.  **優先**: `dev-agent` の安全境界（Git 汚染確認、危険操作ブロック）の実装。
3.  **継続**: メモリシステムの 7層具体化と、日誌生成ロジックの高度化。
