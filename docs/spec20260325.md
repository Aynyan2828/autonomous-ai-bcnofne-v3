# BCNOFNe v3 System Specification (2026-03-25)

## 1. 概要 (Overview)
BCNOFNe v3（ボクノフネ）は、Raspberry Pi 4B 上で動作するコンテナベースの自律型 AI オペレーティングシステムです。2026年3月25日のアップデートにより、Local AI (Ollama) を中心とした安定稼働構成へ移行しました。

## 2. アーキテクチャ (Architecture)
マイクロサービスアーキテクチャを採用し、Docker Compose によって管理されます。

### 主要サービス (Key Services)
- **core**: システムの脳。外部インターフェース（LINE/Discord）との通信、意思決定、タスク管理を担当。
- **ollama**: ローカル LLM サーバー。`Qwen2.5:7b` モデルなどの実行を担当。
- **dev-agent**: 自己改修エンジン。システムのログを監視し、ソースコードの改善案を提案・適用。
- **memory-service**: 7層認知記憶システムの管理。SSD/HDD への記憶の永続化。
- **watchdog**: システム全体の死活監視とフリーズ検知。
- **diary-service**: 航海日誌（Daily Voyage Log）の自動生成。

## 3. LLM 構成 (LLM Configuration)
ハイブリッド構成を採用していますが、プライバシーとオフライン性能を重視し、Local AI を優先します。

- **Primary Model**: Ollama (`qwen2.5:7b`)
- **Fallback/Heavy Model**: OpenAI (`gpt-4o`)
- **Executor**: `llm.executor.LLMExecutor` を通じて全サービスが統一的に LLM を利用。
- **Features**: JSON 補完修復機能、チャンク化要約機能を搭載。

## 4. 安定化設定 (Stability & Performance)
フリーズ防止とリソース最適化のための重要な設定値です。

- **Watchdog 監視間隔**: 21,600秒（6時間）
  - LLM 処理中の誤検知を防ぐため、長期間のヘルスチェック間隔を採用。
- **Ollama Timeout**: 60秒（リトライ 3回）
  - ネットワーク遅延や推論時間に配慮したタイムアウト設定。
- **Health Check**: `core` サービスの `health` コマンドにより、Ollama の生存状態 (ONLINE/OFFLINE) を即時に確認可能。

## 5. ネットワーク & セキュリティ (Network & Security)
- **Zero-trust Access**: Tailscale によるセキュアなリモートアクセスを推奨。
- **Internal Auth**: サービス間通信は `INTERNAL_TOKEN` による認証を実施。
- **External API**: 課金暴走を防ぐ `billing-guard` によるリアルタイム制限。
