# autonomous AI BCNOFNe system v3.3.1 Specification (2026-03-25)

## 1. 概要
本システムは Raspberry Pi 4B (Raspberry Pi OS Bookworm 64bit) 上で動作する自律型 AI OS 「shipOS」である。
AI 人格 **AYN** が OS の核として機能し、多層メモリと自律開発能力を備える。

## 2. コアアーキテクチャ
マイクロサービスアーキテクチャを採用し、Docker コンテナ群で構成される。

### 主要コンポーネント
- **llm (Shared Module)**: 共通 LLM 実行基盤。Ollama (Primary) と OpenAI (Fallback) を統合。
- **memory-service**: 7層メモリモデルに基づく記憶管理。SSD/HDD の階層ストレージ対応。
- **diary-service**: システムログと記憶を統合し、バイリンガルな「航海日誌」と「次期目標」を生成。
- **dev-agent**: 自律改修・デバッグ・アップデートを担うエージェント。SafetyGuard による安全制限。
- **line-gateway / discord-gateway**: 外部通信およびマスター（ユーザー）とのインターフェース。
- **billing-guard**: OpenAI API 等の従量課金コストを監視・遮断。
- **watchdog**: システム全体のヘルス監視。6時間間隔でチェックを行い、フリーズ時は自動再起動。
- **gui / oled-controller**: 物理・仮想ディスプレイへの状態表示。
- **ollama**: ローカル LLM 実行エンジン。メインモデル: `qwen2.5:7b`。

## 3. LLM 実行基盤 (LLM Runtime Policy)
- **Executor**: `llm/executor.py` の `LLMExecutor` に一本化。
- **Fallback Logic**: Ollama が応答しない、または JSON パースに失敗し続けた場合、OpenAI (`gpt-4o-mini`) へ自動フォールバック。
- **Telemetry**: 推論時間、トークン使用量、成功率を統計として保持。

## 4. 7層メモリモデル (Memory Layer)
1. **WORKING**: 作業用一時メモリ。短期タスクや現在の文脈。
2. **EPISODIC**: 日々の出来事。時系列のイベント。
3. **SEMANTIC**: 知識・仕様。システム構成や定数。
4. **PROCEDURAL**: 手順・スキル。特定のタスクの実行方法。
5. **REFLECTIVE**: 内省・学習。失敗からの改善点。
6. **RELATIONAL**: マスターとの関係性。好みや過去の対話傾向。
7. **MISSION**: 中長期目標。システムが目指すべき北極星。

## 5. 自律開発と安全 (Safety Policy)
- **SafetyGuard**: `dev-agent` は実行前に Git ステータスを確認し、未コミットの変更がある場合は停止。
- **Command Filter**: 破壊的コマンド (`rm -rf /` 等) や機密ファイルへのアクセスをブラックリスト方式で制限。
- **Evolution Log**: 全ての自動改修は「進化録」として記録され、Markdown で公開・出力される。

## 6. 運用・監視
- **Watchdog Interval**: 6時間 (21600s)。重い推論中の誤検知を防ぐため長めのインターバルを設定。
- **Billing Limit**: 設定されたコスト上限を超えると、LLMExecutor が即座にリクエストを拒否。

## 7. AYN の振る舞い
- 博多弁を用いたフレンドリーな対話。
- シニアエンジニアとしての確かな技術的自律性。
- 「航海」をメタファーとしたシステム状態の記述。
